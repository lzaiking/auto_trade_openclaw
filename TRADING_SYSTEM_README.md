# Trading System Prototype

这是一个可跑的原型，目标是把用户提出的三件事落地：

1. 给出一套投资逻辑
2. 按这套逻辑在不同时间选股
3. 在最大回撤 20%、总金额 50,000 美元约束下完成自动交易系统并回测

## 文件

- `trading_system.py`：主脚本，负责拉取数据、生成信号、回测、输出订单建议
- `reports/backtest_summary.json`：回测汇总
- `reports/equity_curve.csv`：权益曲线
- `reports/latest_orders.json`：最新一期建议订单

## 策略逻辑

### 1) 投资逻辑
- 标的池：`SPY, QQQ, GOOGL, META, MSFT, AMZN, NVDA, AVGO, AAPL, PLTR`
- 安全资产：`SHY`
- 动量评分：
  - 21 日收益
  - 63 日收益
  - 126 日收益
  - 按 `0.25 / 0.35 / 0.40` 加权
  - 再除以 20 日波动率，偏向风险调整后动量更高的标的
- 趋势过滤：只有价格高于 200 日均线才允许进入候选
- 市场环境过滤：QQQ 与 SPY 都高于 200 日均线时才允许持有风险资产，否则切换到 `SHY`
- 选股：每次调仓选得分最高的 3 只
- 仓位：按逆波动率分配
- 回撤控制：
  - 回撤 > 10%：总风险仓位降到 75%
  - 回撤 > 15%：总风险仓位降到 50%
  - 回撤 > 18%：全部切回 `SHY`

### 2) 不同时间选股
- 每 21 个交易日（月度近似）再平衡一次
- 每次重算评分并更新目标持仓

### 3) 自动交易系统
当前版本默认输出“建议订单”，不直接连真实券商下单，以免误触发真实交易。
如果要实盘接券商，可以在下一版对接：
- Alpaca
- Interactive Brokers
- Tiger / 富途（若有可用 API）

## 运行

```bash
python3 trading_system.py
```
