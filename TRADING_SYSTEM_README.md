# Trading System Prototype

这是一个可跑的原型，目标是把用户提出的三件事落地：

1. 给出一套投资逻辑
2. 按这套逻辑在不同时间选股
3. 在最大回撤 20%、总金额 50,000 美元约束下完成自动交易系统并回测

## 文件

- `trading_system.py`：主脚本，负责拉取数据、生成信号、回测、输出订单建议
- `reports/backtest_summary.json`：回测汇总（含 QQQ buy-and-hold 基准对比）
- `reports/equity_curve.csv`：权益曲线（含策略与 QQQ 基准净值 / 回撤）
- `reports/latest_orders.json`：最新一期建议订单（含建议止损价 / 止盈价）

## 策略逻辑

### 1) 投资逻辑
- 标的池：`SPY, QQQ, GOOGL, META, MSFT, AMZN, NVDA, AVGO, AAPL, PLTR`
- 安全资产：`SHY`
- 动量评分：
  - 21 日收益
  - 63 日收益
  - 126 日收益
  - 按 `0.25 / 0.35 / 0.40` 加权
  - 再除以 20 日波动率，偏向风险调整后动量更高的标的
- 趋势过滤：只有价格高于 200 日均线才允许进入候选
- 市场环境过滤：
  - QQQ 与 SPY 都高于 200 日均线：风险仓位可开满
  - 只有一个高于 200 日均线：保留 `45%` 风险预算，其余停在 `SHY`
  - 两者都低于 200 日均线：切换到 `SHY`
- 选股：每次调仓选得分最高的 4 只
- 仓位：按逆波动率分配
- 回撤控制：
  - 回撤 > 12%：总风险仓位乘以 85%
  - 回撤 > 16%：总风险仓位再乘以 65%
  - 回撤 > 20%：总风险仓位再乘以 35%
- 动态止损 / 止盈：
  - 每天根据 20 日波动率动态计算风控带
  - 止损线：`20日波动率 × 2.3`，并限制在 `10% ~ 22%`
  - 止损线采用**追踪式**（随持仓后的最高价上移）
  - 默认**关闭硬止盈**，避免在强趋势中过早离场

### 2) 不同时间选股
- 每 21 个交易日（月度近似）再平衡一次
- 每次重算评分并更新目标持仓

### 3) 自动交易系统
当前版本默认输出“建议订单”，不直接连真实券商下单，以免误触发真实交易。
如果要实盘接券商，可以在下一版对接：
- Alpaca
- Interactive Brokers
- Tiger / 富途（若有可用 API）

## 运行

```bash
python3 trading_system.py
```

## 备注

- 数据源使用 Stooq 日线 CSV，无需 API key
- 这是一个研究/原型版本，不等于可直接实盘的最终系统
- 下一步建议：加入交易成本、滑点、更多标的、Walk-forward 验证、参数稳健性测试
