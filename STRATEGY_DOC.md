# Trading Strategy Document

## 项目目标

本项目的目标是构建一套可回测、可迭代、后续可连接实盘的美股动量轮动交易系统。在当前阶段，策略优化以两条约束为核心：

- 收益表现优于 `QQQ` buy-and-hold
- 最大回撤小于 `QQQ` buy-and-hold

当前系统使用 50,000 美元初始资金进行回测，风险目标是尽量把最大回撤控制在 20% 左右或以下。

## 选股逻辑

策略核心是 **风险调整后的中期动量排序**。

### 动量评分构成

对每只候选股票，在每个调仓点计算：

- 21 日涨跌幅
- 63 日涨跌幅
- 126 日涨跌幅

并按以下权重加权：

- 21 日：`0.25`
- 63 日：`0.35`
- 126 日：`0.40`

形成原始动量分数：

`raw_momentum = 0.25 * m21 + 0.35 * m63 + 0.40 * m126`

### 风险调整

为了避免高波动标的仅凭短期暴涨冲到前面，原始动量分数还会除以：

- 20 日日收益率标准差

即：

`score = raw_momentum / vol20`

这使得策略更偏好“涨得稳、且持续”的标的，而不是单纯波动很大的股票。

### 趋势过滤

每个候选标的还必须满足：

- 当前价格高于 200 日均线

否则该标的评分直接视为不可入选。

### 最终选股

- 每次调仓，从候选股票中选出得分最高的 `Top 4`
- 仅保留评分大于 0 的标的
- 如果没有符合条件的标的，则全部转入 `SHY`

---

## 仓位分配逻辑

### 逆波动率分配

入选股票之间不是平均分配，而是使用 **逆波动率权重**：

`weight_i ∝ 1 / vol20_i`

含义：

- 波动率较低的标的，分配更高权重
- 波动率较高的标的，分配更低权重

这样可以降低组合集中暴露于单一高波动股票的风险。

### 持仓数量

当前默认持有：

- `Top 4` 风险资产

这是在收益和回撤之间做的折中：

- 持仓过少，集中度太高，回撤容易变大
- 持仓过多，动量被摊薄，收益容易下降

---

## 市场环境过滤（Regime Filter）

策略不在任何时候都满仓持有风险资产，而是根据 `QQQ` 与 `SPY` 的长期趋势状态，决定整体风险预算。

### 判断规则

比较以下两个标的是否高于各自 200 日均线：

- `QQQ`
- `SPY`

### 风险预算规则

#### 1. 两者都强
如果 `QQQ` 和 `SPY` 都在 200 日均线上方：

- 风险预算 = `100%`

即允许正常持有风险资产。

#### 2. 一强一弱
如果只有一个在 200 日均线上方：

- 风险预算 = `45%`

即保留部分风险敞口，其余停放在 `SHY`。

#### 3. 两者都弱
如果 `QQQ` 和 `SPY` 都跌破 200 日均线：

- 风险预算 = `0%`
- 全部转入 `SHY`

### 设计原因

原始版本使用的是更硬的过滤逻辑：

- 只要不满足双确认，就全部退到 `SHY`

这虽然降低了风险，但也导致策略在很多恢复阶段过早离场，损失收益。现在改成分层风险预算后，可以在市场不完全理想但尚未彻底转弱时，保留一定风险暴露。

---

## 回撤控制（Drawdown Governor）

除了市场环境过滤，策略还根据组合自身回撤水平，动态收缩总风险预算。

### 当前规则

设组合从历史高点回撤为 `dd`：

- 如果 `dd > 10%`：总风险预算乘以 `0.85`
- 如果 `dd > 15%`：总风险预算再乘以 `0.60`
- 如果 `dd > 19%`：总风险预算再乘以 `0.30`

### 含义

这不是一次性“清仓/不清仓”的硬阈值，而是渐进式收缩：

- 回撤轻微扩大时，先略降风险
- 回撤继续恶化时，再进一步缩小风险暴露
- 接近风险上限时，大幅降杠杆/降敞口

这样做的目的是：

- 控制组合最大回撤
- 避免在回撤初期就过度保守，导致后续收益恢复能力下降

---

## 出场逻辑

### 1. 动态追踪止损

每只持仓股票都会计算一个基于波动率的止损带。

#### 计算方式

- 先计算该股票近 20 日波动率
- 再乘以系数 `2.3`
- 最终把结果限制在 `10% ~ 22%` 区间内

即：

`stop_pct = clamp(vol20 * sqrt(20) * 2.3, 10%, 22%)`

### 追踪式机制

止损线不是固定的，而是随着持仓后的最高价上移：

`trailing_stop = highest_since_entry * (1 - stop_pct)`

这意味着：

- 股票上涨越多，止损线也会随之抬高
- 一旦价格明显回撤，就会触发退出

### 2. 硬止盈默认关闭

当前版本默认：

- **不启用固定百分比止盈**

原因是此前测试发现，按入场价设置硬止盈，会明显过早截断趋势股上涨，导致组合收益显著下降。

也就是说，当前版本更倾向于：

- 让盈利持仓继续跑
- 用追踪止损保护利润
- 而不是提前用固定止盈把趋势切断

---

## 调仓逻辑

- 调仓频率：每 `21` 个交易日一次
- 可视为近似月度调仓

每次调仓时：

1. 重新计算所有候选标的评分
2. 判断市场环境和当前风险预算
3. 生成目标权重
4. 按目标权重换算成整数股数
5. 输出新的持仓与建议订单

当前实现使用整股近似，不考虑碎股。

---

## 订单生成逻辑

系统当前不直接连券商下单，而是输出建议订单到：

- `reports/latest_orders.json`

订单内容包括：

- 标的代码
- 买卖方向
- 数量
- 估算价格
- 估算金额
- 目标权重
- 建议止损比例
- 建议止损价格

如果未来重新启用硬止盈，也可以扩展输出建议止盈价。

## 回测指标

系统当前输出的核心指标包括：

- 结束权益
- 总收益率
- CAGR（年化收益率）
- 年化波动率
- Sharpe Ratio
- 最大回撤

并同时输出 `QQQ buy-and-hold` 的对比指标，包括：

- QQQ 总收益率
- QQQ CAGR
- QQQ 年化波动率
- QQQ Sharpe
- QQQ 最大回撤

同时给出布尔比较结果：

- 是否跑赢 QQQ 总收益
- 是否跑赢 QQQ CAGR
- 是否最大回撤低于 QQQ

## 当前策略特征总结

这套策略本质上是：

**“趋势过滤 + 风险调整动量选股 + 分层市场环境过滤 + 动态追踪止损 + 渐进式回撤控制”**

它不是高频交易，也不是均值回归，而是偏中周期的趋势/轮动框架。

### 优点

- 逻辑清晰，可解释性强
- 回测层面已实现“收益打过 QQQ、回撤小于 QQQ”
- 风险控制比单纯 buy-and-hold 更主动
- 容易继续扩展到交易成本、滑点、实盘接口

### 当前不足

- 仍未加入交易成本与滑点
- 仍未做 walk-forward / out-of-sample 稳健性测试
- 仍未做再入场优化（止损后可能要等到下次调仓）
- 股票池当前仍偏科技成长，风格暴露较集中

---

## 后续优化方向

下一阶段建议优先做：

1. **加入交易成本与滑点**
   - 让回测更接近实盘

2. **加入 walk-forward / 滚动参数验证**
   - 避免策略只对当前样本有效

3. **优化止损后的再入场机制**
   - 避免被洗出后长时间空窗

4. **扩展股票池或加入行业 ETF**
   - 降低单一风格过度集中风险

5. **接入券商 API 前的执行层设计**
   - 下单限制
   - 风险检查
   - 订单确认
   - 交易日志
